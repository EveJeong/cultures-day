## 시스템 한 줄 요약

여러 기기(빔프로젝터·참가자 폰·운영자 폰)가 **하나의 게임 상태**를 실시간으로 공유한다.
운영자가 운영 페이지에서 상태를 바꾸면, 모든 진행 페이지가 즉시 갱신된다.

## 기술 스택

| 항목 | 선택 |
| --- | --- |
| 프론트엔드 | React + Vite + TypeScript, Tailwind CSS |
| 실시간 동기화 | Firebase Firestore (`onSnapshot`) |
| 이미지 저장 | AWS S3 + presigned URL(3일). 업로드/조회 URL 발급용 **Lambda + API Gateway 서명서버** |
| 배포 | Vercel (공개 URL) |
| 운영 보호 | `admin` 계정 로그인 (인증은 `01-auth-design.md`) |
| 참가자 접속 | 진행 페이지에 QR 코드 노출 |

> **하이브리드 구성**: 실시간 상태는 Firebase, 퀴즈 이미지는 S3(+Lambda 서명), 프론트는 Vercel.
> 브라우저는 자격증명 없이 S3에 직접 업로드할 수 없어, presigned URL을 발급하는 최소 백엔드가 필요하다.

## 게임 목록 (9종, 예시 프리셋)

게임은 관리자가 CRUD로 정의한다(`02-admin-design.md` 기능 9). 두 축 — **진행 방식
(engineType: quiz/prompt/none)** 과 **점수 방식(scoringType: quiz/rank/increment/free)** — 을
자유 조합한다. 아래는 "기본 9게임 불러오기" 프리셋 초기값.

| # | 게임 | engineType | scoringType | 배점 |
| --- | --- | --- | --- | --- |
| 1 | 퀴즈 (과자이름·초성·확대샷·노래·영화) | quiz | quiz | 문제당 10 |
| 2 | 릴레이 몸으로 말해요 | prompt | increment | 정답당 N |
| 3 | 이어서 그리기 | prompt | increment | 정답당 50 |
| 4 | 라이어게임 | none | free | 총배점 내 운영자 배분(검증) |
| 5 | 과자 맞추기 (미각) | none | increment | 개당 20 |
| 6 | 윷놀이 + 훈민정음 | none | rank | 200 / 100 / 50 |
| 7 | 보드게임 (루미큐브·로보77·할리갈리) | none | rank(×3 라운드) | 라운드별 100 / 50 / 10 |
| 8 | 릴레이 게임 (공기·물병·제기·딱지·탁구공) | none | rank | 200 / 100 / 50 |
| 9 | 병뚜껑 컬링 | none | free | 미정(총배점 옵셔널) |

> 진행 엔진(quiz q1~q3 / prompt w1~w4 / none 일반)과 점수 방식을 분리해, 게임마다 컨트롤을
> 따로 만들지 않고 메타데이터만으로 운영/진행 화면이 알맞게 렌더링된다. 미정 배점/시간은 데이터만 수정.

## 핵심 원칙

1. **단일 진실 공급원(SSOT)**: 모든 상태는 Firestore에 있고, 화면은 그 구독 결과를 그릴 뿐이다.
2. **유형 기반 일반화**: 9개 게임을 4개 점수 유형으로 환원한다.
3. **되돌리기 가능**: 모든 점수 변경은 로그로 남겨 실수 시 취소할 수 있다.
4. **현장 견고성**: 네트워크가 잠시 끊겨도 Firestore SDK의 오프라인 캐시로 버틴다.
